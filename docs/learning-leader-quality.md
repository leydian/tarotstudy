# 학습 리더 자연화 품질 운영 가이드

## 목표
학습 리더 메시지를 사람이 읽기에 자연스럽고, 실제 행동으로 옮길 수 있도록 유지한다.

## 점검 축 (각 1~5점)
- 자연스러움: 문장이 어색하지 않고 조사/호응 오류가 없다.
- 행동가능성: 즉시 실행 가능한 행동 지시가 포함된다.
- 중복도: 동일 표현 반복이 과도하지 않다.
- 페르소나 일관성: 학습 코치 톤이 유지된다.

합격 기준:
- 전체 평균 4.0 이상
- 3.0 미만 항목 비율 10% 이하

## 자동 점검
명령:
- `npm run qa:learning-leader`

동작:
- 고정 평가셋(30개) + A/B 실험군 각각 호출
- 메시지 규칙 검증(질문 포함, 행동 포함, 금지 패턴, 중복도)
- 점수 집계 및 실패 항목 출력

## 수동 리뷰 루프
명령:
- `npm run qa:learning-leader -- --write-review`

산출물:
- `tmp/learning-leader-review.md`

운영:
- 주 1회 20개 샘플 수동 리뷰
- 각 샘플에 대해 "어색한 원인-수정 문장-규칙 반영 여부"를 기록

## A/B 실험 운용
- API draw 요청에 `experimentVariant` (`A`/`B`)를 명시해 강제 측정 가능
- 지표 비교:
  - 복기 저장률
  - 복기 메모 길이
  - 다음 리딩 재방문률
