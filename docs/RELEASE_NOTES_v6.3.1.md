# Release Notes v6.3.1 (2026-02-28)

## 목적
리딩 데이터 생성은 정상인데 결과 UI가 화면에 보이지 않는 문제를 해결했습니다.
문제의 본질은 API/모델이 아니라 스프레드 레이아웃 계산이었습니다.

## 이슈 요약
- 증상: 카드가 펼쳐진 뒤에도 "운명의 리포트"가 보이지 않거나 매우 아래로 밀려 보이지 않음
- 발생 조건: 좌표 범위가 큰 스프레드(특히 5장 커리어 스프레드)
- 영향: 리딩이 실패한 것으로 오해될 수 있음(실제 데이터는 생성됨)

## 원인 분석
`TarotMastery.tsx`에서 카드 좌표 렌더 배율을 고정값(0.5)로 사용하고,
상단 스프레드 영역 높이를 `maxAbsY` 기반으로 크게 잡아 결과 메시지 영역이 하단으로 밀렸습니다.

즉, 데이터/네트워크 문제가 아니라 **레이아웃 비례 계산 문제**였습니다.

## 변경 상세
- 파일: `apps/web/src/pages/TarotMastery.tsx`

### 1) 스프레드 렌더 설정 함수 추가
- `getSpreadRenderConfig(spread)` 신설
- 계산 값:
  - `scale`: 스프레드 좌표 범위에 따라 자동 조정
    - 최소 0.28, 최대 0.5
  - `areaHeight`: 상단 영역 높이 자동 조정
    - 최소 280px, 최대 420px

### 2) 카드 좌표 배치 동적 스케일 적용
- 기존: `left/top = pos * 0.5` 고정
- 변경: `left/top = pos * renderConfig.scale`
- 효과: 좌표가 큰 스프레드도 화면 안에 수렴

### 3) 상단 스프레드 영역 높이 제한
- 기존: 내용 기반으로 계속 증가 가능
- 변경:
  - `minHeight = renderConfig.areaHeight`
  - `maxHeight = 48vh`
- 효과: 결과(messages/result) 영역이 항상 시야에 남도록 보장

## 사용자 체감 개선
- 카드가 큰 배치여도 결과 리포트가 즉시 보임
- "리딩이 안 뜬다"는 오해를 유발하던 레이아웃 장애 제거
- 모바일/작은 화면에서도 스크롤 부담 감소

## 검증
- `npm run build --prefix apps/web` 통과
- 커리어 5장/관계 7장 등 큰 스프레드에서 결과 영역 노출 확인

## 리스크 및 후속
- 현재 계산식은 좌표 기반 휴리스틱이므로 신규 스프레드 추가 시 시각 튜닝이 필요할 수 있음
- 후속으로 스프레드별 권장 배율 preset을 둘 수 있음
