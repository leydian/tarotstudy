# Changelog

## [2026-02-28]

### 지능형 상황 인지 리딩 엔진 도입 (Reading V3 - Context-Aware V5.2)

#### 변경 사항
- **질문 객체 추출(Entity Extraction) 로직 구현**:
    - "A 아니면 B", "A 할까 B 할까" 등의 양자택일 질문에서 핵심 객체(Entity)를 자동으로 추출하는 NLP-lite 정규식 적용.
    - 추출된 객체를 리딩 서사(예: "[걷기]의 길을 살펴보니")와 지침(Action)에 직접 매핑하여 답변의 직결성 향상.
- **질문 무게(Gravity) 감지 및 페르소나 톤 조절**:
    - 일상적 키워드(커피, 메뉴, 이동수단 등) 및 질문 길이를 바탕으로 질문의 무게를 'Light'와 'Heavy'로 분류.
    - 'Light' 질문 시 과장된 운명적 표현(예: "운명의 거대한 파도")을 억제하고, 친근하고 실용적인 사서의 톤으로 자동 전환.
- **상황적 의미 합성(Contextual Meaning Synthesis)**:
    - 카드 고유의 원소 상징(검=이성, 컵=감정, 펜타클=효율 등)을 추출된 질문 객체와 동적으로 결합.
    - 예: '검' 카드와 '이동수단'이 만나면 "이성적인 판단이 필요한 이동"으로 해석하는 등 질문 도메인에 맞춤화된 리딩 생성.
- **양자택일(Binary) 로직 근본적 리팩토링**:
    - 시간 순서 기반의 서사 구조를 완전히 탈피하고, 두 선택지를 대조군으로 배치하여 장단점을 명확히 비교 분석.
    - 종합 분석 단계에서 단순히 점수 합산이 아닌, 두 객체 간의 기운을 직접 비교하여 최선의 선택을 제안.

### 양자택일 리딩 서사 로직 개선 (Reading V3 - Binary Logic)

#### 변경 사항
- **병렬적 서사 구조 도입**:
    - `isBinary` 상태일 때, 시간 순서(과거→미래)가 아닌 병렬적 선택(A vs B)으로 카드를 해석하도록 로직 수정.
- **비교 분석형 종합 판정 적용**:
    - 두 선택지의 기운을 직접 비교하여 사용자에게 최선의 길을 제안하는 문구로 변경.

### AI 리딩 엔진 고도화 (Reading V3 - Narrator V5.1)

#### 변경 사항
- **서사적 위치(Position) 설명 개선**:
    - 실제 스프레드 위치 라벨(예: 잠재적 재능, 숨겨진 기회)을 문장에 자연스럽게 녹여내도록 수정.
- **문법적 자연스러움 강화**:
    - **조사 처리 헬퍼(`getJosa`) 확장**: '을/를', '은/는' 처리를 추가하여 문맥에 맞는 정확한 한국어 조사 선택.
    - **조언 문구 연결 최적화**: 명령형 조언과 서사 사이의 문법적 단절 해결.
