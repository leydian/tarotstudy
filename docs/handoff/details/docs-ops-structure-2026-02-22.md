## 이번 정리에서 반영한 운영 이슈
1. 다크 테마 가독성 이슈
- 증상: 일부 UI 요소에서 텍스트 대비 부족
- 반영: 변수 기반 대비값 재조정, 컨트롤 텍스트 색 강제

2. 레이아웃 체감 부족 이슈
- 증상: 테마만 바뀌고 구성 변화가 약하게 체감됨
- 반영: 앱 셸/페이지 공통 레이아웃 패턴 재편, 스프레드 세부 정보 카드화

3. 주별 운세 라벨 회귀 이슈
- 증상: 월~일 체계 요구 후 레거시 라벨 재노출
- 반영: 데이터 확인 + 회귀 테스트 추가로 재유입 차단

## 문서 운영 규칙
- 신규 기능/품질개선 완료 시:
  - 메인: 핵심 변화 3~7줄 요약
  - 인덱스: 해당 영역 링크 추가/정리
  - 세부: 변경 이유, 파일, 검증, 리스크를 기록
- 큰 UI 작업은 스크린샷 근거와 함께 "가독성 개선 항목"을 명시
- 회귀 버그는 반드시 테스트 문서/테스트 파일 링크를 같이 남김
