# Release Notes v6.3.5 (2026-02-28)

## 목적
AGENTS/CLAUDE 가이드를 실제 웹 앱 개발 절차에 적용해, 구현 품질을 개인 역량이 아닌 팀 표준으로 고정합니다.

## 변경 요약
- 개발 실행 표준 문서 신설
- README 문서 내 개발 워크플로우 링크 추가
- PR 템플릿에 범위/성공조건/사실-가정 분리 섹션 추가

## 변경 상세

### 1) 개발 워크플로우 표준 문서 추가
- 파일: `docs/DEVELOPMENT_WORKFLOW.md`
- 핵심 내용:
  - 작업 시작 시 대상/유형/산출물/성공기준 명시
  - 구현 중 국소 변경 원칙과 실패 대응 루프(증상→가설→검증→수정→재검증)
  - API/네트워크 변경 시 필수 확인 항목(`apiUsed`, `fallbackUsed`, `meta.fallbackReason`)
  - 코드 변경 시 문서 동기화 규칙
  - 웹 앱 전용 체크포인트(질문 의도 vs 스프레드 일치, 체감 품질 확인)

### 2) README 문서 링크 보강
- 파일: `README.md`
- 변경:
  - `📚 상세 문서` 목록에 `🛠 개발 워크플로우 표준` 링크 추가

### 3) PR 템플릿 운영 기준 확장
- 파일: `.github/pull_request_template.md`
- 추가 섹션:
  - `Scope & Success`
  - `Facts / Assumptions / Open Questions`
- 의도:
  - 구현 전 범위/완료 조건 고정
  - 리뷰 단계에서 사실과 추정을 분리해 의사결정 품질 향상

## 기대 효과
- 요청 범위 이탈 및 불필요한 리팩터링 감소
- 실패 시 무작위 재시도 대신 가설 기반 수정 문화 정착
- PR 리뷰에서 요구사항 누락/모호성 조기 발견
- 코드와 문서의 동기화 누락 감소

## 운영 가이드
- 모든 코드 변경 PR은 템플릿 신규 섹션을 채워야 합니다.
- 문서/설정 계약 변경 시 릴리스 노트 또는 변경 이력 동시 갱신을 유지합니다.
- 네트워크 장애/모델 실패가 있는 작업은 반드시 로그 근거와 fallback 상태를 보고에 포함합니다.
